Why a different approach?

what is wrong with the status quo?

Software Engineering has become knowledgeware which is the assembly of existing components and solutions to provide a new and improved solution. This makes it incredibly easy to create new applications with basic features fast. As it result it can make it harder to manage complexity as the app grows over time.

And when this complexity grows the constant marketing of the industry with new tools continues to convince teams that they have picked the wrong tool. And a desire to re-write the current solution using new tools and technologies gets sold as the solution to the complexity problem.

The problem with this process is that it is a cycle and often if you do not introduce the correct ingredients to transform instead of replace you end right back in the same place. The mission of hyper63 is to stop this cycle.

About hyper63

hyper63 was founded in August 2020, by Tom Wilson, and its mission is to improve software engineering by using training and consulting to teach and mentor a growth mindset and design patterns that focus on future-proofing software and creating happy software teams.

There are a lot of great tactics on how to create high-performing teams and processes, how to organize teams and how to organize code. All of these tactics are great, but they are not silver bullets, to effectively transform, you need more than tactics, you need to transform the mind of the team, you need to get and keep everyone on the same page and commit to disipline and focus.
